# AGENTS.md

Guidance for AI coding agents working in this repository.

## Project Snapshot

Helix Scroll is a vanilla HTML/CSS/JavaScript single-page interactive gallery.
There is no package manager, bundler, framework, or build step.

The app renders an immersive Three.js helix of Ahri skin cards over a fixed
canvas backdrop and a styled page shell. Scroll position, pointer movement,
canvas dragging, and card hover all influence the 3D scene.

## How To Run

Open `index.html` directly in a browser, or serve the folder with any static
file server:

```bash
npx serve .
```

```bash
python -m http.server 8080
```

Three.js is loaded from the import map in `index.html`:
`https://cdn.jsdelivr.net/npm/three@0.165.0/build/three.module.js`.
Three add-ons are imported through the `three/addons/` import-map prefix.

## Important Files

- `index.html`: document shell, import map, canvas, fixed nav/seek/music/caption
  UI, dot and scroll-hint containers, and audio tag.
- `script.js`: all Three.js scene setup, postprocessing, generated card
  textures, particles, scroll mapping, hover raycasting, drag input, seek/dot UI,
  and music controls.
- `styles.css`: page background, fixed panels, canvas visibility, responsive
  behavior, dot indicator, scroll hint, music states, and reduced-motion CSS.
- `assets/`: local images, background art, and `Legends Never Die.mp3`.
- `CLAUDE.md`: additional project notes that may overlap with this file.

## Runtime Architecture

The 3D scene is built entirely in `script.js`. Rendering goes through
`EffectComposer` with `RenderPass` and `UnrealBloomPass`; the scene also uses a
small generated canvas environment map for reflections.

Scene hierarchy:

```text
scene
root
  particleRoot
  galleryRoot
    spineRoot
    panelRoot
```

`particleRoot` contains shader-driven point clouds for the outer particle helix
and inner dust. `spineRoot` contains the central torus-knot vertebrae, helix
rails, halos, and glow columns. `panelRoot` contains the card meshes and each
card's additive hover-glow child mesh.

`scrollProgress` ranges from `0` to `1` and determines:

- `activeCardIndex`
- `scrollRotation`
- `targetGalleryY`
- dot indicator state
- caption opacity
- `document.documentElement.dataset.activeCard`

The animation loop is capped near 60 fps. It eases gallery rotation, gallery Y
offset, particle alignment, camera position, card opacity, card scale, hover
glow, and pointer tilt every frame, then calls `composer.render()`.

## Cards And Assets

Card images are listed in the `imageFiles` array in `script.js`.

To add, remove, or reorder cards:

1. Put the image in `assets/`.
2. Update `imageFiles`.
3. Check card titles from `getCardTitle()` and `splitTitleLines()`.
4. Check seek-list display names from `buildSkinList()`.
5. Verify scroll spacing, focus behavior, dot count, hover glow, and mobile
   framing.

Card textures are not static image files. They are generated with Canvas 2D in
`makePanelTexture()` and `drawPanel()`, then used as `THREE.CanvasTexture`
materials. The card canvas composites the image, tint overlays, vignette,
gloss, deterministic grain, title text, index label, and rounded border.

Hover glow is also canvas-generated by `getGlowTexture()` and reused by all
card glow meshes.

## Interaction Notes

- Page scroll controls card focus.
- Pointer movement adds subtle camera and root tilt.
- Dragging on the canvas scrolls the page.
- The drag axis locks after a small movement threshold.
- Horizontal drag is amplified to make left/right swipes navigate cards.
- The seek panel links scroll to the first several cards.
- Dot indicators reflect the current card and are hidden on narrower screens.
- Raycasting against `cardMeshes` sets `hoveredCardIndex`; hovered cards push
  outward, scale up, show an additive border glow, and brighten `hoverGlowLight`.
- `prefers-reduced-motion: reduce` freezes continuous shader/floater motion and
  disables CSS animations, while preserving scene rendering and scroll behavior.

## Music Notes

The audio element is defined in `index.html` and controlled from `script.js`.
Music remains paused until the user clicks the music panel Play button or
starts scrolling the page.
Do not add autoplay or unrelated first-gesture retry behavior outside those
controls.
If playback is blocked after the Play button is clicked, the panel is marked
with `needs-interaction`.
When the page is hidden or loses focus, any playing music should pause and
should not resume from focus alone.

The music button uses play/pause symbol text, updates `aria-pressed` and
`aria-label`, and toggles `is-playing` on `.music-panel`.
The volume slider initializes audio at 50% volume and should preserve the
user-selected level across play, pause, scroll-start playback, and focus-loss
pauses.

## Styling Guidelines

- Keep the first screen as the actual immersive experience, not a landing page.
- Preserve the fixed canvas backdrop, fixed UI panels, and scroll-driven first
  screen unless the requested change specifically calls for a layout shift.
- Keep responsive behavior aligned between `resize()` in `script.js` and CSS
  breakpoints in `styles.css`. JS currently switches at `560px` and `980px`;
  CSS switches at `780px` and `560px`.
- Avoid introducing a build tool or dependency unless the user explicitly asks.
- Keep text legible over the animated scene at desktop and mobile sizes.
- The background art comes from `assets/background.png`; do not replace it with
  purely decorative gradients unless requested.

## Debugging

`window.helixDebug` exposes live values:

```js
window.helixDebug
```

Expected fields:

- `cardCount`
- `activeCardIndex`
- `scrollProgress`

Use this in the browser console while checking scroll behavior.

`document.documentElement.dataset.activeCard` is also updated as a two-digit
card number for styling/debugging.

## Verification Checklist

For visual or interaction changes, verify:

- `index.html` loads without console errors.
- The canvas renders a nonblank Three.js scene.
- Scrolling advances through all cards.
- Dragging on the canvas scrolls the page.
- Hovering cards updates the glow and does not break drag scrolling.
- Seek panel links scroll to the expected cards.
- Dots match the active card count and active state.
- Music play/pause button updates text, `aria-pressed`, and panel state.
- Mobile widths around `560px` still frame the gallery and panels cleanly.
- Reduced motion mode does not break scene rendering.

## Change Discipline

- Keep edits focused to the requested behavior.
- Do not rename assets unless all references are updated.
- When changing CSS/JS loaded by `index.html`, update the cache-busting query
  string if the change needs to be visible after deploy.
  Current query strings are `styles.css?v=fox-spirit-13` and
  `script.js?v=card-detail-2`.
- Be careful with existing uncommitted changes. Treat them as user work unless
  told otherwise.
